2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 12.10 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, Version 12.10 offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its 25-year history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • New for version 12.00
    • AddonSoftware Install and Upgrade Wizard: New Graphical Interfaces
      • Install Wizard: For new Addon installations, the Install Wizard saves time by consolidating the Copy aon Directory, Update addon.syn File, and Update sync XML Files utilities into one seamless step.
      • Upgrade Wizard: For upgrading existing Addon Installations, the Upgrade Wizard consolidates the Copy aon Directory, Update addon.syn File, and Update sync XML Files utilities, merging in data from the prior installation to preserve settings. It also installs a new copy of Barista and runs the Auto-Sync process.
    • Order/Invoice Entry - A new field in the Sales Order Processing parameter record provides control over whether or not users are warned or focus is forced onto the Totals tab to re-evaluate Freight and Discount amounts when changing existing line items.
    • GL Report Format Listing - Prints a listing of heading and detail information for a financial report. It can be used for reference or as a worksheet for maintaining a financial report.
    • GL Posting Control by Firm and Module - Module parameter tables have been enhanced to provide control over GL postings.
    • GL Fiscal Calendar auto-adjust February for leap year - Added new parameter to indicate whether a period end date of February 28 will be automatically adjusted to the 29th on leap years.
    • Demo/Eval "nag" - Uses a Barista Auto-launch process to warn when Addon is launched from the BASIS product download location. Addon should only be launched/run from the BASIS product download location for demos/evals.
    • Inventory Costing Report - Provides an exploded listing of Bills of Materials, including costs rolled up from the lowest level component to the highest assembly.
    • Inventory Costing Update - Updates Inventory Master and Bill of Material costs by first updating component costs and then using those costs to update higher levels through to the highest assembly.
    • Bill of Materials Module
      • Inventory Production Entry – Allows input of items that are assembled or produced according to the BOM Master file.
      • Order Production Entry – Allows automatic selection of items from the Customer Sales Order file for Inventory Production Entry.
      • Bill of Materials Packing List – Used to help with assembly, quality control, and boxing of an item for shipment.
      • Materials Where Used Listing – Prints a list of Items and in which Bills they are referenced.
      • Operation Where Used Listing – Prints a list of Operation Codes and in which Bills they are referenced.
      • Indented Bill of Materials Report – A listing of all levels of a Bill including the cost of Materials, Operations, and Subcontracts.
      • Bill of Materials Detail Listing – Prints detail information about each Bill selected.
      • Bill of Materials Costing Report - An exploded list detailing each selected Bill and its costs along with the details of each Bill's sub-assemblies.
      • Bill of Materials Masters – Defines the header information about a BOM.
        • Operations – Allows entry of detail information about each Operation that takes place in the BOM.
        • Materials – Allows entry of detail information about each Item that is used in the BOM.
        • Totals – Allows for inquiry into the various costs associated with the BOM.
        • Packing List – see Bill of Materials Packing List above.
        • Delete – Removes the BOM and all of its various components from the system.
        • Copy Bill – Allows quick duplication of a BOM to a new Bill number.
        • Availability – Allows entry of a quantity needed of the finished BOM, then displays the quantity available for each item used in that BOM based on requirements defined.
        • Comments – Allows entry of comments associated with the BOM.
        • Subcontracts – Allows entry of information of any outside contracting work associated with the BOM.
      • Bill of Materials Costing Update - Modifies Inventory Unit Costs to reflect the costs reported on the Bill of Materials Costing Report.
      • Bill Detail Exception Report - Lists linkage anomalies between material component lines and operation or subcontractor lines (e.g., a subcontract line linking to a comment instead of to an item.
      • Component Item Replacement - Replaces a specified component, for a bill or range of bills, with another component based on effective and obsolete dates.
  • Preview for version 13.00
    • AddonSoftware Shop Floor Control
      • Work Order Entry
        • Header Information - Defines the Work Order type, status, and category as well as the item or activity the Work Order describes. Also, allows linking the Work Order to a customer's Sales Order.
        • Material Requirements - The list of materials required for the Work Order. The list is automatically populated when the Work order is for an item defined in the Bill of Materials module. Data include quantity, scrap factor, cost as well as a multiplier and divisor for refining unit definition.
        • Operation Requirements - Defines the Operations necessary to produce an item--including hours and costs related to queue, setup, move, and production.
        • Schedule Work Order - Scheduling options include Backwards from an estimated completion date, Forward from an estimated start date, and Manual methods.
        • Release Work Order - Commits material necessary for the production of a work order. Optionally displays shortages. Allows for flagging a work order for Traveler printing and Pick List printing.
        • Job Status - Reports status information for a work order--including Required Date, Completed Quantity, and Standard versus Actual setup and run times.
        • Transaction History - Reports unit, rate quantity completed, and cost information for a date range of transactions associated with a work order--optionally filtered by transaction type (Materials, Operations, or Subcontracts). See Work Order Transaction History Report for the batch version of this report.
        • Cost Summary - Displays the accumulated costs of transactions against a work order, broken down by operation code, material issued, and subcontracts. Totals, variance amounts, and variance percentages are shown.
        • Comments Maintenance - Allows addition of comments for a work order. Useful for storing engineering notes, special handling instructions, etc.
        • Subcontract Requirements - Defines subcontract information including rate, units, costs, and optionally, Purchase Order.
      • Dispatch Inquiry - Displays information about a selected Operation to assist in work flow scheduling decisions. The data can be sorted by the various columns, and can be filtered by date range, priority, and Work Order status.
      • Create Purchase Requisitions - Generates non-stock Purchase Requisition lines for unattached subcontract requirements within a selected range of Work Orders.
      • Materials Issues Entry - Allows entry of items and quantities to be issued toward a Work Order. While generally materials issues will be generated from the materials on the pick list, you can issue any material to any open work order
      • Materials Issues Register - Reports pending materials issues.
      • Materials Issues Update - Relieves Inventory of the item quantities listed on the Materials Issues Register and generates a Work Order Transaction reflecting the action.
      • Work Order Header Report - Lists information from Work Order headers based on a variety of filter and sorting options. Includes Work Order Type, Start and Completion dates, Closed Quantities and Values, Date Opened, and, if applicable, Customer, and Sales Order.
      • Labor Efficiency Analysis Report - Lists information for the analysis of the efficiency of your labor force by operation. It will review performance of each operation in comparison to the standards defined.
      • Cost Variance Analysis Report - Lists information for the analysis of standard versus actual costs. Can be run using a variety of filter and sorting options. It is useful in adjusting standard labor hours, tracking the performance of shop labor and production management, and in analyzing the sources of labor variances.
      • Date Analysis Report - Lists information for the analysis work orders for an estimated start or complete date, actual start dates, or last active date. Can be run using a variety of filter and sorting options.
      • WO Transaction History Report - Lists transaction information for Work Orders based on a variety of filter and sorting options. May include data such as labor hours and charges, material issues, and subcontract receipts. See the Transaction History task under Work Order Entry for the on-demand version of this report.
      • Work Order Job Status Summary - Summarizes Work Orders--listing Required and Last Activity dates, standards and actuals for operations, direct labor, overhead and materials, as well as variances.
      • Type Codes Maintenance - Allows maintenance of various codes to define Work Order Types which tell the system in what category a work order belongs, how to close the work order, to which general ledger accounts the information will be updated, etc.
      • Operation Codes Maintenance - Allows maintenance of various codes to define Work Order Operations when the Bill of Materials module is not installed or interfaced with the Shop Floor module.
      • Employees Maintenance - Allows maintenance of various codes to define employee-related information when the Payroll Processing module is not installed or interfaced with the Shop Floor module.
      • Create Shop Floor Calendar - Defines the initial shop floor calendars for operations. The schedule is used by the Work Order Entry task to determine how many hours of a particular operation are available for each day.
      • Shop Floor Calendar Maintenance - Allows changes to the Shop Floor Calendars created via the Create Shop Floor Calendar task.
      • Copy Shop Floor Calendar - Provides a convenient mechanism for copying Shop Floor Calendars from one operation to another.
      • Work Order Parameters - Allow tailoring of the module to the specific needs of a business. Includes optional interfacing of the Shop Floor module with the Bill of Materials, Order Processing, and Payroll modules.

  • For a list of continued improvements in all modules, refer to the "fixed issues" link on the BASIS product download page.
3. KNOWN ISSUES(top of page)
  • BBj version 12.10 or higher and the Barista Application Framework version 12.10 or higher are both required to run AddonSoftware version 12.10. Earlier versions may produce unexpected results
  • When running under Linux, the KDE desktop environment must be used.
4. SUPPORT(top of page)

If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

Technical support for version 12.10 is available via e-mail at addon-developer@addonsoftware.com.

When submitting a technical support question, please provide the following information:
  1. The product serial number.
  2. The name of the product and the product's version number.
  3. A complete description of the problem, including the product error and the TCB(10) value, if available.
  4. A screen shot of the problem.
Other pertinent information includes:
  1. The operating system name, JRE, level and version.
  2. Is the error consistently reproducible, or is it sporadic?
  3. Is the error isolated to a specific machine or user?
  4. Is this a new installation or has the system been in place and operating normally?
  5. Is the problem network related/specific?
